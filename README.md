# 강민수 · Minsu Kang

DACON Data Science Team — 대회파트 매니저
국민대학교 AI빅데이터융합경영학과

---

학부 때부터 데이콘 공모전에 자주 참가했습니다. 그중 유전체 품종 분류 대회(716팀)에서 1위를 했고, 법원 판결 예측에서는 공동 1위와 스코어 동점이었지만 제출이 조금 늦어 2등으로 확정됐습니다. HD현대 AI Challenge는 예선 2위로 본선에 올라 현대 본사에서 발표했습니다.

2025년 6월부터 11월까지 슈어소프트테크 AX응용기술팀에서 AI 개발자 인턴으로 일했습니다. 해양 도메인 LLM을 적응시키는 일, AI 에이전트를 시나리오·데이터셋 기반으로 검증하는 일, 현대자동차 자율주행 ML 검증 가이드라인을 정리하는 일을 맡았습니다.

2025년 12월부터는 데이콘에서 일합니다. 참가자로 뛰던 플랫폼의 운영자가 된 셈인데, 덕분에 문제가 어떻게 만들어지고 평가되는지 양쪽에서 볼 수 있게 됐습니다. 한 가지 모델을 오래 파는 것보다, 문제 정의 → 모델링 → 평가 → 운영을 하나로 잇는 쪽에 관심이 더 있습니다.

## Now — 데이콘에서 하고 있는 일

- 여러 LLM을 같은 기준 위에서 비교하기 위한 평가 시스템 설계·구현
- NIA·한국교통안전공단 등 공공 데이터 기반 외부 AI 과제 수행
- Vibe-coding 기반 대회 플랫폼 **Daker** 기여 — 기능 개선, 버그 대응
- 사내 업무용 에이전트 **Grim** 개발
- 대회 기획·운영, B2B·B2C PM 업무

## Selected work

**[Genomic_Data_Breed_Classification](https://github.com/Minsu5452/Genomic_Data_Breed_Classification)** — 716팀 중 1위 · 3인 팀장
고차원 구조화 데이터 문제를 앙상블로 해결했습니다. 1위 결과가 이어져 *2023년 제2차 실용 인공지능 학술대회(AAiCON 2023)*에 논문이 기재되고 현장 발표까지 진행됐습니다. 관련 자료는 [AAiCON2023](https://github.com/Minsu5452/AAiCON2023) 저장소에 정리해 두었습니다.

**[Court_Judgment_Prediction](https://github.com/Minsu5452/Court_Judgment_Prediction)** — 2위 · 5인 팀장
장문 판결문을 다룬 NLP 대회. 1위 팀과 스코어가 동점이었고, 제출 시각이 조금 늦어 2등이 됐습니다.

**[HD_Hyundai_AI_Challenge](https://github.com/Minsu5452/HD_Hyundai_AI_Challenge)** — 예선 2위(330팀), 본선 6위(11팀)
조선·해운 산업 데이터를 다룬 예측 대회. 예선은 팀장, 본선은 팀원으로 참가해 HD현대 본사에서 오프라인 발표를 했습니다. 5등까지 수상이라 아슬아슬하게 비켜난 경험이기도 합니다.

**[DACON_LLM_Evaluation](https://github.com/Minsu5452/DACON_LLM_Evaluation)** — 데이콘
플랫폼 위에서 여러 LLM을 동일 기준으로 비교해야 할 때 쓸 평가 체계를 정리했습니다. 평가 항목 정의, metric 조합, 파이프라인 설계가 핵심입니다.

**[KT_Agent_Verification](https://github.com/Minsu5452/KT_Agent_Verification)** — 슈어소프트테크
AI 에이전트의 행동을 시나리오·데이터셋으로 검증. 실패 케이스를 유형화해 품질 관리 관점으로 정리했습니다.

**[Marine_LLM](https://github.com/Minsu5452/Marine_LLM)** — 슈어소프트테크
해양 도메인에 LLM을 적응시키는 작업. 도메인 데이터 확보, retrieval 설계, 품질 점검까지 묶어서 다뤘습니다.

**[Hyundai_Autonomous_Guideline](https://github.com/Minsu5452/Hyundai_Autonomous_Guideline)** — 슈어소프트테크
현대자동차 자율주행 ML 검증 가이드라인. 검증 관점의 기준·리스크·프로세스를 문서로 구조화했습니다.

**[Receipt_Data_NER](https://github.com/Minsu5452/Receipt_Data_NER)** — 개인
OCR → 자동 라벨링 → BIO tagging → BERT NER 학습까지 혼자 붙여 본 end-to-end 프로젝트.

**[L-point_Customer_Analytics](https://github.com/Minsu5452/L-point_Customer_Analytics)** — 팀 프로젝트
고객 행동 데이터에서 시작해 세그먼트와 마케팅 전략까지 이어 본 분석 프로젝트.

전체 저장소는 [여기](https://github.com/Minsu5452?tab=repositories)에서 볼 수 있습니다.
